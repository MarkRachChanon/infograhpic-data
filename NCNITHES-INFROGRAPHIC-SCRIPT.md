## 🔄 SECTION 3: Workflow การใช้งานหลัก

### Headline
> **5 ขั้นตอน — สร้าง รายงาน อนุมัติ ติดตาม ส่งออก**

### Flowchart (แนวนอน 5 กล่อง)

```
┌──────────────┐   ┌──────────────┐   ┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│  1. แต่งตั้ง   │ → │  2. นิเทศ +  │ → │  3. รายงาน    │ → │  4. อนุมัติ    │ → │  5. ส่งออก   │
│   ครูนิเทศ    │   │   ปฏิบัติงาน  │   │   ในระบบ      │   │   3 ขั้น        │   │   PDF        │
│  (Admin ทำ)  │   │   (จริง)     │   │   (ครูทำ)    │   │   (หน./บริหาร)│   │   (Admin/ผู้ใช้)│
└──────────────┘   └──────────────┘   └──────────────┘   └──────────────┘   └──────────────┘
```

### รายละเอียดแต่ละขั้น

#### ขั้นที่ 1 — Admin แต่งตั้งครูนิเทศ
- เลือก **ระดับชั้น × ชั้นปี × ห้อง × สาขา × ครู**
- 1 ครู ดูแลได้หลายห้อง · 1 ห้อง มีครูร่วมได้
- ระบบ track ความครบสัปดาห์อัตโนมัติ

#### ขั้นที่ 2 — ครูออกไปนิเทศจริง
- ไปสถานประกอบการ / นิเทศออนไลน์
- ถ่ายรูปกิจกรรม · เก็บความเห็นจากสถานประกอบการ · เก็บข้อมูลครูฝึก

#### ขั้นที่ 3 — ครูบันทึกรายงานในระบบ
**ข้อมูลที่ต้องกรอก:**
- ภาคเรียน / วันที่นิเทศ / สัปดาห์ที่ (1-17)
- เวลาเริ่ม-สิ้นสุด · สาขา · รูปแบบนิเทศ (Onsite/Online) · วิธีเดินทาง
- **ครูร่วมนิเทศ** (เลือกจาก dropdown — ลดการรายงานซ้ำ)
- ข้อมูลสถานประกอบการ + **ครูฝึก/ตำแหน่ง/เบอร์** (บังคับ)
- ระดับ/ชั้นปี/ห้อง/นักศึกษา · รายวิชา · ปัญหา/แนวทาง
- รูปกิจกรรม (อย่างน้อย 2 รูป)

#### ขั้นที่ 4 — ผ่านการอนุมัติ 3 ขั้น (ดูใน Section 4)

#### ขั้นที่ 5 — ส่งออก PDF
- **บันทึกข้อความ** ต่อรายงาน — มีลายเซ็นครบทุกระดับ + วันที่อนุมัติ
- **รายงานสรุป** ระดับสาขา / รายสัปดาห์ / ความก้าวหน้าต่อครู

### Visual แนะนำ
- Arrow ใหญ่เชื่อมระหว่าง 5 กล่อง
- ใต้แต่ละกล่องมีไอคอน + ข้อความ 2-3 บรรทัด
- ใช้สีเดียวกันทั้งแถวเพื่อแสดง flow ต่อเนื่อง

## ✅ SECTION 4: ขั้นตอนการอนุมัติ (3-Stage Workflow)

### Headline
> **อนุมัติ 3 ขั้น — โปร่งใส ตรวจสอบได้**

### Timeline แนวนอน

```
[ ครูส่ง ] → [ Stage 1 ] → [ Stage 2 ] → [ Stage 3 ] → [ ✅ สมบูรณ์ ]
              หัวหน้าแผนก/   หัวหน้างาน    ผู้บริหาร/
              หัวหน้าภาค                   ผอ.
```

### ตารางขั้นอนุมัติ

| ขั้น | ใครอนุมัติ | เงื่อนไข |
|---|---|---|
| **Stage 1: รอหัวหน้าแผนก/ภาค** | • ปวช./ปวส. → `is_department_head` ในสาขาเดียวกัน<br>• ปริญญาตรี → `is_faculty_head` ในสาขาเดียวกัน | ต้องอยู่ในสาขาเดียวกับรายงาน (admin ข้ามเงื่อนไขนี้ได้) |
| **Stage 2: รอหัวหน้างาน** | `is_work_head` | cross-department |
| **Stage 3: รอผู้บริหาร** | • ผู้อำนวยการ / รองผู้อำนวยการ (role)<br>• หรือคนที่ติด flag `is_executive` | — |
| **✅ อนุมัติสมบูรณ์** | — | ผ่านทั้ง 3 ขั้น |

### กรณีพิเศษ
- 🔴 **ส่งกลับแก้ไข**: ในแต่ละขั้น คนที่อนุมัติ "กดส่งกลับ" ได้ → ระบบ reset ทุกขั้น เริ่มใหม่
- ⚠️ **เปลี่ยนสถานะที่อนุมัติสมบูรณ์**: หัวหน้างาน / ผู้บริหาร / admin กด "ไม่อนุมัติ (เปลี่ยนสถานะ)" ได้อีก หากพบปัญหาภายหลัง
- 💬 **เหตุผลที่พบบ่อย (chip)**: คลิกครั้งเดียวเพิ่ม คลิกซ้ำถอน — กันการกรอกซ้ำ

### Visual แนะนำ
- Timeline แนวนอนพร้อม checkmark สำหรับขั้นที่ผ่านแล้ว
- หลอด progress 3 ขั้น (สีเหลือง → เขียว เมื่อ approve)
- ไอคอนคนแต่ละหมวกประจำขั้น

## 🤝 SECTION 5: ครูร่วมนิเทศ (ลดความซ้ำซ้อน)

### Headline
> **ไปนิเทศด้วยกัน — รายงานครั้งเดียว นับร่วมกัน**

### หลักการ
- ครู A เป็นเจ้าของรายงาน + กดเลือกครู B เป็น "ครูร่วมนิเทศ"
- ครู B จะเห็นรายงานในแท็บ **"ห้องที่ฉันรับผิดชอบ"** + นับเป็นสัปดาห์ที่ทำแล้ว (ถ้าห้องตรงกับ assignment ของครู B)
- ครู B แก้ไขไม่ได้ → ดูได้อย่างเดียว

### เงื่อนไขการนับ
- ✅ นับเครดิตให้ครูร่วม **เฉพาะ** ถ้า `ระดับชั้น × ชั้นปี × ห้อง` ตรงกับ assignment ของครูร่วม
- ❌ ไม่นับ ถ้าครูร่วมไม่ได้รับการแต่งตั้งห้องนั้น (กันการ "เก็บแต้ม" ผิดห้อง)
- ✅ ตัด **บันทึกข้อความ "ครูยังไม่รายงาน"** ออก — ถ้าเป็นครูร่วมในสัปดาห์นั้น

### Visual แนะนำ
- ภาพครู 2 คนยืนนิเทศนักศึกษาร่วมกัน
- ลูกศรไปยังรายงาน 1 ฉบับ
- ตัวอย่างหน้าจอ widget search + checkbox + badge
